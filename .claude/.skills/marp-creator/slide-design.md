# スライド設計（ステップ2）

整理済みMarkdownからスライド設計書を作成する。レイアウトや情報量を調整する中間成果物。

## 入力

| 項目 | 説明 |
|------|------|
| `01_organized_[資料名].md` | organize-contentの出力 |
| 発表時間（任意） | 分単位（例：5分、15分） |
| テーマ（任意） | kanagawa-wave / classmethod |

## 発表時間とスライド枚数の目安

| 時間 | 枚数 |
|------|------|
| 5分 | 5〜8枚 |
| 10分 | 10〜15枚 |
| 15分 | 15〜20枚 |
| 30分 | 20〜30枚 |

## 出力

`02_slide-design_[資料名].md`

### ファイル構造

```markdown
# スライド設計書: [資料タイトル]

**テーマ:** kanagawa-wave
**想定枚数:** X枚 / **想定時間:** Y分

## スライド構成一覧

| No. | タイトル | レイアウト | 備考 |
|-----|----------|-----------|------|
| 1 | [タイトル] | lead | 表紙 |
| 2 | [セクション] | section | 第1章 |
| 3 | [内容] | 基本 | 箇条書き3項目 |

## スライド詳細

### スライド1: [タイトル]
**レイアウト:** lead / **ページネーション:** false
- タイトル: [資料タイトル]
- サブテキスト: [日付・所属など]

### スライド2: ...
```

## レイアウト選択基準

### kanagawa-wave テーマ

| レイアウト | 使用場面 |
|-----------|----------|
| lead | 表紙（1枚目） |
| section | 大きなトピックの切り替わり |
| no-header | フルスクリーンで内容を見せたい場合 |
| （基本） | テキスト・箇条書き中心 |

### classmethod テーマ

| レイアウト | 使用場面 |
|-----------|----------|
| title | 表紙（1枚目） |
| section | 大きなトピックの切り替わり |
| image | 図表をメインにしたいスライド |
| content-image-right/left | 図表とテキストの横並び |
| column-layout | 複数項目の比較 |
| small-text | 情報量が多い（最終手段） |

## ワークフロー

1. 整理済みファイルを読み込み、テーマを確認
2. 発表時間からスライド枚数を決定
3. 内容をスライド単位に分割（1スライド1メッセージ）
4. 各スライドにレイアウトクラスを割り当て
5. 情報量を調整（箇条書き3〜5項目/スライド）
6. `02_slide-design_[資料名].md` を出力
