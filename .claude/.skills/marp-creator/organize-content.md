# 内容整理（ステップ1）

散在するメモや箇条書きを、論理的に整理された構造化Markdownファイルに変換する。この段階ではMarpの記法やスライド枚数を意識せず、伝えたい内容の構造化に集中する。

## 入力

| 項目 | 説明 | 形式 |
|------|------|------|
| メモファイル | 整理対象の内容 | Markdownファイルまたはテキスト |
| 資料の目的 | 発表、勉強会、説明資料など | テキスト |
| 対象者 | 社内メンバー、エンジニアなど | テキスト |
| 出力先 | ファイルの配置場所（任意） | パス |

## 出力

`01_organized_[資料名].md` を生成する。

### ファイル構造

```markdown
# [資料タイトル]

## 資料の概要

**目的:** [資料の目的を1〜2文で記載]
**対象者:** [想定する読者・聴衆]
**作成日:** [YYYY-MM-DD]

---

## [メイントピック1]

[導入文や概要を地の文で記載]

### [サブトピック1-1]

[内容を説明]

**図表:** [ここに○○の図を挿入予定]

---

## 補足・メモ

[追加情報や検討事項があれば記載]
```

## ワークフロー

1. 入力ファイルを読み込み、目的・対象者を把握
2. 主要トピックを洗い出してグループ化
3. 論理的な順序に並び替え（課題→解決策→効果 等）
4. 見出しレベルを h1〜h3 で整理（h4以降は使わない）
5. 箇条書き（3項目以上）と地の文（2項目以下）を使い分け
6. 図表が必要な箇所を `**図表:**` で明示
7. `01_organized_[資料名].md` を出力

## 判断基準

| 項目 | 基準 |
|------|------|
| トピックの粒度 | 1トピックで3〜5分程度説明できる内容 |
| 見出しレベル | h1〜h3まで |
| 箇条書き項目数 | 3〜7項目程度 |
| 1段落の長さ | 3〜5文程度 |
